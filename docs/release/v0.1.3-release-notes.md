# Anlyx v0.1.3

Anlyx v0.1.3 makes the project usable as a local Live Workspace developer tool.

The real frontend keeps running on its own localhost port, Anlyx runs beside it on port `4777`, and the Live Workspace focuses on the API request caused by the user's latest action instead of behaving like a full network log console.

## Install

```bash
npm install -D anlyx
npx anlyx init
npx anlyx dev
```

## Highlights

- Workspace Mode keeps the real frontend at `frontend.baseUrl` while Anlyx serves local runtime assets, live event ingest/stream, report data, and the full-page Workspace from port `4777`.
- `AnlyxDevOverlay` installs the capture helper only during Next.js development builds.
- The Live Workspace prioritizes the latest click, submit, or key-triggered API request.
- Background auth, session, health, and polling requests are recorded but separated from the main flow.
- The matched backend flow is rendered through Summary, Timing, Diagram, and Evidence Inspector views.
- The compact capture badge opens the Workspace without blocking the host app.
- React SPA and manual integration remain available through the raw capture script.
- README and release assets should show the full-page Live Workspace rather than a drawer mock.
- Packaging now resolves workspace dependencies correctly for published npm packages.

## Compatibility

Deep Support:

- Spring Boot backend endpoint and flow scanning
- Next.js App Router page discovery and capture

Basic Support:

- OpenAPI endpoint import
- Manual frontend URLs
- Manual capture script injection for non-Next React apps

Not included in v0.1:

- FastAPI, Express, NestJS, or React Router Deep Support
- Java agent runtime tracing
- OpenTelemetry tracing
- Mermaid, PNG, or SVG export
- LLM-generated flow summaries

## Upgrade Notes

- Use `anlyx@0.1.3` or newer.
- Older `anlyx@0.1.0` and `anlyx@0.1.1` builds are not recommended.
- Source package files use pnpm workspace dependencies inside the monorepo, but published tarballs are verified so workspace dependencies are resolved to real npm versions.
