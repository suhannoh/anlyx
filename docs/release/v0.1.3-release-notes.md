# Anlyx v0.1.3 Release Notes Draft

## Summary

Anlyx v0.1.3 is the next patch release candidate because the `v0.1.2` git tag already exists on `origin`. It is intended for normal `npm install anlyx` usage after the approved pnpm-based publish. The release focuses on making Anlyx usable as an injected local developer tool: the real app stays on its own port, Anlyx runs behind it, and the Flow Drawer focuses on the request caused by the user's latest action.

## Highlights

- Three-command onboarding target: `npm install -D anlyx@0.1.3`, `npx anlyx init`, `npx anlyx dev`.
- Inject Mode keeps the real frontend at its own `frontend.baseUrl` and runs the Anlyx runtime on port `4777`.
- Next.js helper `AnlyxDevOverlay` injects the overlay only during development.
- Action-first Flow Drawer prioritizes the latest click, submit, or key-triggered API request.
- Background auth/session/health/polling requests are recorded but kept separate from the main flow.
- Main backend flow is rendered as a React Flow diagram with fit/reset controls.
- Compact, draggable Anlyx launcher uses the project icon and avoids blocking the host app.
- React SPA/manual integration remains available through the raw overlay script.
- Spring Boot + Next.js App Router remain the v0.1 Deep Support targets.
- Packaging fixes for the `0.1.0` workspace dependency issue and the `0.1.1` CLI entrypoint issue.
- README and release assets now show the actual Anlyx Flow Drawer component instead of a static mock.

## Compatibility

Deep Support:

- Spring Boot backend endpoint and flow scanning
- Next.js App Router page discovery and capture

Basic Support:

- OpenAPI endpoint import
- Manual frontend URLs
- Manual overlay script injection for non-Next React apps

Not included in v0.1:

- FastAPI, Express, NestJS, or React Router Deep Support
- Java agent runtime tracing
- OpenTelemetry tracing
- Mermaid, PNG, or SVG export
- LLM-generated flow summaries

## Verification Checklist

Before publishing:

- `corepack pnpm install`
- `corepack pnpm typecheck`
- `corepack pnpm lint`
- `corepack pnpm test`
- `corepack pnpm format`
- `corepack pnpm -r build`
- `git diff --check`
- package dry-runs from `docs/release/v0.1-release-runbook.md`

Local smoke target:

- Zup frontend at `localhost:3000`
- Anlyx runtime at `localhost:4777`
- Overlay launcher appears without blocking the app
- User action requests become the main drawer flow
- Background `/api/account/me` style auth checks stay quiet unless selected
- Drawer remains usable with opacity, drag, resize, language, and diagram controls
- README demo GIF is generated from `docs/assets/readme/anlyx-demo.html`

## Upgrade Notes

- Use `anlyx@0.1.3` after the approved publish.
- Do not use `anlyx@0.1.0`; it was published with unresolved `workspace:*` dependencies.
- Do not use `anlyx@0.1.1`; the published CLI entrypoint can exit before command execution.
- Use `corepack pnpm publish`, not direct `npm publish`, when releasing packages from this workspace.
