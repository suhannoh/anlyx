# Anlyx v0.1.6-beta.0 Release Notes

## Positioning

Anlyx v0.1.6-beta.0 is a local viewer for reviewing AI-agent-authored codebase
maps.

It is not positioned as an automatic deep analyzer for every framework. The
recommended workflow is:

```txt
Ask your coding agent to write anlyx.project.json
-> validate the Project JSON
-> import it locally
-> open http://localhost:4777
-> review Pages, Map, Overview, Capabilities, and JSON
```

## What's Included

- Project JSON validation for `schemaVersion` `0.2.0` and `0.3.0`.
- Source reference validation for authored file paths, line references, and
  symbols, with validation reports written during validate/import.
- Optional coverage metadata for detected, modeled, and unmodeled project
  surfaces so partial analysis can be shown honestly.
- Local import and viewer startup through the `anlyx` CLI.
- Pages workspace for page stories, requests, flows, evidence, and unknowns.
- Layered Map workspace for agent-authored architecture relationships.
- Overview and Capabilities surfaces for human-friendly project understanding.
- Raw JSON inspection surface for reviewing exactly what the agent authored.
- Viewer trust summaries for source issues, coverage status, and validation
  report warnings.
- Evidence labels that keep source-matched, observed, measured,
  agent-inferred, not-proven, and unknown claims distinct.
- Public demo and docs site for previewing the product workflow.
- Agent prompt helpers:
  - `npx anlyx prompt init`
  - `npx anlyx prompt refresh`
- Official `anlyx refresh` agent shortcut for updating an existing
  `anlyx.project.json` from changed files without rewriting the whole project
  model.

## Who This Beta Is For

- Developers using Codex, Claude Code, Cursor, or another coding agent to
  understand a repository.
- Teams that want an inspectable artifact instead of one-off AI chat answers.
- Reviewers who need to see which project facts are backed by evidence and
  which are inferred or still unknown.
- Portfolio/project owners who want to present page, request, architecture, and
  evidence relationships from a local project snapshot.

## Known Limits

- Anlyx does not automatically understand every framework by itself.
- The quality of the viewer depends on the quality of the authored
  `anlyx.project.json`.
- Large real-world projects need more examples and stress testing.
- Validation error UX is still CLI-first and should become more guided.
- Timing remains optional and should only appear when real measurement data is
  authored.

## Recommended First Run

Paste this into a coding agent inside the project you want to map:

```txt
Install Anlyx with npm install -D anlyx@beta.
Read https://github.com/suhannoh/anlyx and the Project JSON agent guide.
Analyze this repository and create anlyx.project.json.
Include pages, requests, flows, architecture nodes, evidence, overview, and capabilities when you can support them.
Do not invent facts. Mark uncertain links as agent-inferred, not-proven, or unknown.
Run:
  npx anlyx validate anlyx.project.json
  npx anlyx import anlyx.project.json
  npx anlyx dev
Open http://localhost:4777 and report what is source-matched, inferred, or still unknown.

Remember this shortcut for later:
When I type "anlyx refresh", update the existing anlyx.project.json from current repository changes instead of recreating everything.
```
