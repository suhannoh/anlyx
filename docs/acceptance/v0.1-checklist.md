# v0.1 Acceptance Checklist

## Purpose

This checklist defines how Anlyx v0.1 implementation completion will be judged. An implementation Agent MUST report against this checklist.

Note: `npx anlyx ...` is the intended published-package command shape. Before npm publication, acceptance is verified through the local workspace CLI entrypoint.

## CLI

- [x] `npx anlyx init` creates `anlyx.config.ts`.
- [x] `npx anlyx scan` creates `.anlyx/report-data.json`.
- [x] `npx anlyx dev` starts the local Anlyx runtime on `localhost:4777` by default.
- [x] Inject Mode keeps the real frontend at `frontend.baseUrl` and supports `AnlyxDevOverlay` from `anlyx/next`.
- [x] Inject Mode treats `localhost:4777` as a background runtime/status surface while the real app remains at `frontend.baseUrl`.
- [x] The standalone debug viewer remains reachable at `localhost:4777/_anlyx/viewer`.
- [x] The injected overlay automatically focuses user-action requests, while page-load/background/health/polling requests remain selectable secondary events.
- [x] The injected overlay opens the main drawer for React SPA/manual frontend user-action `fetch` calls without requiring the Next.js helper.
- [x] The injected overlay shell supports opacity, resize, drag positioning, and Korean/English shell labels.

## Config

- [x] Spring Boot + Next.js config matches `docs/contracts/config-contract.md`.
- [x] OpenAPI-only config matches `docs/contracts/config-contract.md`.
- [x] OpenAPI-only backend plus manual frontend is supported through `ManualFrontendAdapter`.
- [x] `sampleParams` controls dynamic route capture.
- [x] `storageState` is passed to Playwright capture when configured.

## Spring Adapter

- [x] Collects `GET /api/public/benefits/{id}`.
- [x] Creates Controller node.
- [x] Creates Service node.
- [x] Creates Repository node.
- [x] Creates `benefits` Database node.
- [x] Distinguishes Main Flow and Sub Flow.
- [x] Includes confidence values.
- [x] Emits unknown or warning data when analysis is incomplete.

## OpenAPI Adapter

- [x] Imports endpoints from `paths`.
- [x] Extracts method, path, tags, request schema, and response schema.
- [x] Does not create Controller, Service, Repository, or Database nodes for OpenAPI-only backends.
- [x] Treats OpenAPI as Basic Support only.

## Next Adapter

- [x] Collects `app/**/page.tsx`, `page.jsx`, `page.ts`, and `page.js`.
- [x] Does not collect layouts, loading files, errors, not-found files, components, hooks, utilities, tests, stories, or API routes as pages.
- [x] Dynamic routes remain template routes.
- [x] Dynamic routes with `sampleParams` can be captured.
- [x] Dynamic routes without `sampleParams` become `captureStatus = "pending"`.

## Manual Frontend Adapter

- [x] `frontend.type = "manual"` selects `ManualFrontendAdapter`.
- [x] `ManualFrontendAdapter` converts each `frontend.urls` entry into one `PageStoryboard`.
- [x] Manual page storyboards omit `filePath` unless an explicit mapping is added.
- [x] Manual page storyboards can be captured by `CaptureAdapter`.

## Capture

- [x] Captures page screenshot segments with 1440x900 default viewport.
- [x] Collects API calls during capture.
- [x] Keeps failed pages in output.
- [x] `/admin/benefits` can be represented as `captureStatus = "failed"` with `Login required`.

## Data Contract

- [x] Output conforms to `docs/contracts/data-contract.md`.
- [x] `.anlyx/report-data.json` is a `ScanResult`.
- [x] Fixture expected JSON matches the Data Contract.

## UI

- [x] Endpoint Map is visible.
- [x] Page Storyboard is visible.
- [x] Right Inspector is visible.
- [x] Main Flow and Sub Flow are visually distinct.
- [x] Low-confidence or unknown analysis is visible.
- [x] Default UI follows Clean Light tokens.
- [x] Dark Replay demo direction is possible without making the whole product dark-only.

## Replay Lite

- [x] Play, pause, restart, loop on/off, and main-flow-only controls exist.
- [x] Request dot travels through `EndpointFlow.mainPath` in forward order.
- [x] Response dot returns through `EndpointFlow.mainPath` in reverse order.
- [x] v0.1 fixtures do not require explicit `response` edges.
- [x] Sub Flow replay is not implemented in v0.1.
- [x] Advanced timeline is not implemented in v0.1.

## Fixture

- [x] `fixtures/spring-next-sample/expected/endpoints.json` matches `Endpoint[]`.
- [x] `fixtures/spring-next-sample/expected/flows.json` matches `EndpointFlow[]`.
- [x] `fixtures/spring-next-sample/expected/pages.json` matches `PageStoryboard[]`.
- [x] `fixtures/spring-next-sample/expected/report-data.json` matches `ScanResult`.
- [x] `fixtures/spring-next-sample/expected/report-data.json` embeds endpoints.json, flows.json, pages.json without losing required fields or metadata.
- [x] Fixture data includes the Zup benefit detail flow.
- [x] Fixture data includes `/admin/benefits` failed capture.

## Scope Guard

- [x] No FastAPI Deep Support.
- [x] No Express Deep Support.
- [x] No NestJS Adapter.
- [x] No React Router Deep Support.
- [x] No static HTML export.
- [x] No Mermaid export.
- [x] No PNG or SVG export.
- [x] No Java Agent runtime tracing.
- [x] No LLM flow summary.
- [x] No 3D UI.

## Release Packaging

- [ ] `corepack pnpm -r build` produces package `dist` outputs without source test files in npm tarballs.
- [ ] `corepack pnpm --filter anlyx pack --dry-run` includes the CLI bin entry, package metadata, README, LICENSE, and built `dist` files.
- [ ] Packed package metadata does not retain `workspace:*` dependency ranges.
- [ ] Local tarball install can run `npx anlyx --help` and `npx anlyx init` without publishing to npm.
- [ ] `0.1.3` is published with `corepack pnpm publish`, not `npm publish`, so workspace dependencies are converted before registry upload.
- [ ] Published CLI entrypoint does not use top-level await and can run `--help`, `init --force`, and `scan --skip-capture`.
