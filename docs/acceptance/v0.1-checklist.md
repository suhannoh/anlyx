# v0.1 Acceptance Checklist

## Purpose

This checklist defines how Anlyx v0.1 implementation completion will be judged. An implementation Agent MUST report against this checklist.

## CLI

- [ ] `npx anlyx init` creates `anlyx.config.ts`.
- [ ] `npx anlyx scan` creates `.anlyx/report-data.json`.
- [ ] `npx anlyx dev` opens a local UI on `localhost:4777` by default.

## Config

- [ ] Spring Boot + Next.js config matches `docs/contracts/config-contract.md`.
- [ ] OpenAPI-only config matches `docs/contracts/config-contract.md`.
- [ ] `sampleParams` controls dynamic route capture.
- [ ] `storageState` is passed to Playwright capture when configured.

## Spring Adapter

- [ ] Collects `GET /api/public/benefits/{id}`.
- [ ] Creates Controller node.
- [ ] Creates Service node.
- [ ] Creates Repository node.
- [ ] Creates `benefits` Database node.
- [ ] Distinguishes Main Flow and Sub Flow.
- [ ] Includes confidence values.
- [ ] Emits unknown or warning data when analysis is incomplete.

## OpenAPI Adapter

- [ ] Imports endpoints from `paths`.
- [ ] Extracts method, path, tags, request schema, and response schema.
- [ ] Does not create Controller, Service, Repository, or Database nodes for OpenAPI-only backends.
- [ ] Treats OpenAPI as Basic Support only.

## Next Adapter

- [ ] Collects `app/**/page.tsx`, `page.jsx`, `page.ts`, and `page.js`.
- [ ] Does not collect layouts, loading files, errors, not-found files, components, hooks, utilities, tests, stories, or API routes as pages.
- [ ] Dynamic routes remain template routes.
- [ ] Dynamic routes with `sampleParams` can be captured.
- [ ] Dynamic routes without `sampleParams` become `captureStatus = "pending"`.

## Capture

- [ ] Captures page screenshot segments with 1440x900 default viewport.
- [ ] Collects API calls during capture.
- [ ] Keeps failed pages in output.
- [ ] `/admin/benefits` can be represented as `captureStatus = "failed"` with `Login required`.

## Data Contract

- [ ] Output conforms to `docs/contracts/data-contract.md`.
- [ ] `.anlyx/report-data.json` is a `ScanResult`.
- [ ] Fixture expected JSON matches the Data Contract.

## UI

- [ ] Endpoint Map is visible.
- [ ] Page Storyboard is visible.
- [ ] Right Inspector is visible.
- [ ] Main Flow and Sub Flow are visually distinct.
- [ ] Low-confidence or unknown analysis is visible.
- [ ] Default UI follows Clean Light tokens.
- [ ] Dark Replay demo direction is possible without making the whole product dark-only.

## Replay Lite

- [ ] Play, pause, restart, loop on/off, and main-flow-only controls exist.
- [ ] Request dot travels from page to backend to database.
- [ ] Response dot returns to the page.
- [ ] Sub Flow replay is not implemented in v0.1.
- [ ] Advanced timeline is not implemented in v0.1.

## Fixture

- [ ] `fixtures/spring-next-sample/expected/endpoints.json` matches `Endpoint[]`.
- [ ] `fixtures/spring-next-sample/expected/flows.json` matches `EndpointFlow[]`.
- [ ] `fixtures/spring-next-sample/expected/pages.json` matches `PageStoryboard[]`.
- [ ] `fixtures/spring-next-sample/expected/report-data.json` matches `ScanResult`.
- [ ] Fixture data includes the Zup benefit detail flow.
- [ ] Fixture data includes `/admin/benefits` failed capture.

## Scope Guard

- [ ] No FastAPI Deep Support.
- [ ] No Express Deep Support.
- [ ] No NestJS Adapter.
- [ ] No React Router Deep Support.
- [ ] No static HTML export.
- [ ] No Mermaid export.
- [ ] No PNG or SVG export.
- [ ] No Java Agent runtime tracing.
- [ ] No LLM flow summary.
- [ ] No 3D UI.
