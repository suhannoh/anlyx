# Anlyx v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Break Anlyx v0.1 implementation into reviewable PRs that preserve the locked Spring Boot + Next.js App Router scope.

**Architecture:** v0.1 uses a TypeScript monorepo with Core contracts, adapters, capture, CLI, and UI packages. Data flows from config loading to backend/frontend adapters, capture, report aggregation, and a local UI that reads `ScanResult`.

**Tech Stack:** Node.js, TypeScript, pnpm workspace, Vitest, ESLint, Prettier, Playwright, React, React Flow, Dagre, Motion, Tailwind CSS, shadcn/ui-compatible component conventions.

---

## 1. Implementation Principles

- v0.1 Deep Support MUST be limited to Spring Boot + Next.js App Router.
- OpenAPI MUST provide Basic Support only.
- Manual frontend URLs MUST be handled by `ManualFrontendAdapter`.
- Replay Lite response movement MUST use `EndpointFlow.mainPath` reverse traversal.
- `report-data.json` MUST embed the same endpoint, flow, and page objects as the split expected JSON files.
- Failed, unknown, and pending states MUST remain visible in data and UI.
- Any contract change required during implementation MUST update `docs/contracts/*` before implementation changes.
- Fixture expected output is the baseline for scanner, aggregation, UI, and acceptance checks.
- Each PR SHOULD be small enough to review independently and MUST keep excluded v0.1 features out of scope.
